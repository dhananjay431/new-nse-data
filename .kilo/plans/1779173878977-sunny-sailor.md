# Plan: Implement Sticky First Column in Tables

## Overview
Implement sticky positioning for the first column in all table views (IndexTablePage, AllIndicesPage, and ETFPage) while keeping other columns scrollable. This will improve user experience when viewing wide tables by keeping the identifying column (usually symbol/name) visible during horizontal scrolling.

## Requirements
- First column in all tables should be sticky (fixed) horizontally
- Other columns should remain scrollable
- Maintain existing table functionality (sorting, filtering, etc.)
- Ensure visual consistency across all table components
- Work properly in both light and dark modes

## Implementation Steps

### 1. Update CSS for Sticky Columns
Add CSS classes to make the first column sticky in all table implementations:
- Position the first column with `position: sticky` and `left: 0`
- Add appropriate z-index to ensure the sticky column appears above other content
- Set background color to prevent content from showing through during scrolling

### 2. Modify DataTable Component
Update the shared DataTable component (`frontend/src/components/DataTable.jsx`):
- Add a new prop to enable/disable sticky first column functionality
- Apply sticky styling to the first `th` and `td` elements in each row
- Ensure proper handling of dark mode backgrounds

### 3. Update IndexTablePage Component
Modify the table rendering function in `frontend/src/pages/IndexTablePage.jsx`:
- Apply sticky column styling to the first column
- Ensure the symbol column (typically the first) remains visible during horizontal scroll
- Maintain existing sorting and filtering functionality

### 4. Update AllIndicesPage Component
Modify the table rendering in `frontend/src/pages/AllIndicesPage.jsx`:
- Apply sticky column styling to the first column
- Handle both table view and heatmap view appropriately
- Ensure consistent styling with other components

### 5. Test Implementation
- Verify sticky behavior works in all table views
- Check compatibility with sorting and filtering
- Ensure proper display in both light and dark modes
- Test with various screen sizes

## Code Changes

### CSS Changes
Add the following CSS classes to support sticky columns:
```css
/* In src/index.css */
.sticky-first-col {
  position: sticky;
  left: 0;
  z-index: 1;
}

.sticky-first-col-header {
  position: sticky;
  left: 0;
  z-index: 2;
  box-shadow: 1px 0 0 #e2e8f0; /* Add shadow to separate from scrollable columns */
}

.dark .sticky-first-col {
  background-color: #1e293b; /* dark:bg-slate-800 */
}

.dark .sticky-first-col-header {
  background-color: #334155; /* dark:bg-slate-700 */
  box-shadow: 1px 0 0 #334155; /* Match dark mode */
}
```

### Component Changes
1. Update `DataTable.jsx` to support sticky first column as an optional feature
2. Modify table rendering functions in `IndexTablePage.jsx` to apply sticky styling
3. Update table rendering in `AllIndicesPage.jsx` similarly
